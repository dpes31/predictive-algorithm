# Algorithm Specification v1.0.0

상태: **FROZEN / Gate 0 승인 완료**  
승인일: 2026-06-30  
변경 권한: 사용자 명시 승인 + 모델 버전 증가 필수

---

## 1. 목적

과거 로또 6/45 결과와 향후 누적 데이터를 이용해 다음 회차의 6개 번호 조합 5세트를 생성한다. 이 시스템은 단순 출현빈도 추천기가 아니라 다음 네 가지 가설을 동시에 비교하는 동적 확률모형이다.

- M0: 무신호·균등 무작위
- M1: 과거 신호의 지속
- M2: 과거 신호의 반전·평균회귀
- M3: 구조변화·레짐 전환

모델은 신호가 확인되지 않을 때 복잡한 패턴을 억지로 만들지 않고 M0으로 복귀해야 한다.

---

## 2. 기본 정의

- 번호 집합: `N = {1, 2, ..., 45}`
- 한 회차 조합: `S ⊂ N`, `|S| = 6`
- 전체 조합 수: `C(45,6) = 8,145,060`
- 회차 t의 실제 당첨 조합: `Y_t`
- 회차 t 예측 시점까지 사용 가능한 데이터: `D_{t-1}`

보너스번호는 저장·표시하지만 v1.0.0 예측 목표에는 포함하지 않는다.

---

## 3. M0 균등 무작위 기준모형

```text
P0(S) = 1 / C(45,6)
```

M0는 제거할 수 없는 살아 있는 경쟁모형이다. 비균등모형이 미래 검증에서 기준선을 이기지 못하면 M0의 가중치가 증가해야 한다.

---

## 4. 입력 특징

번호 i, 회차 t에 대한 특징 벡터:

```text
x_i,t = [
  long_frequency,
  recent_10_frequency,
  recent_30_frequency,
  recent_52_frequency,
  gap_since_last_seen,
  pair_interaction,
  trend,
  local_entropy,
  change_point_score
]
```

주의:

- 특징의 존재가 미래 예측력을 의미하지 않는다.
- 빈도·엔트로피·시차·번호쌍·변화점은 먼저 무작위성 게이트와 상태 추정에 사용한다.
- 특정 특징의 효과 방향을 미리 고정하지 않는다.

---

## 5. 상태·레짐별 동적 번호 점수

상태 r에서 번호 i의 점수:

```text
η_i,t,r = α_i,r + Σ_k β_k,t,r · x_i,k,t
```

계수는 시간에 따라 변화할 수 있다.

```text
β_k,t,r = β_k,t-1,r + ν_k,t,r
```

따라서 동일한 특징은 시점과 상태에 따라 다음처럼 작동할 수 있다.

- `β > 0`: 긍정신호
- `β < 0`: 부정신호
- `β ≈ 0`: 예측정보 없음

이 구조는 과거의 긍정신호가 미래에 부정신호로 전환되거나 그 반대가 되는 현상을 반영한다.

---

## 6. 상태별 조합확률

상태 r에서 조합 S의 확률:

```text
P_r,t(S)
= exp[
    Σ_(i∈S) η_i,t,r
    + Σ_(i<j, i,j∈S) γ_i,j,t,r
  ] / Z_r,t
```

- `η_i,t,r`: 번호별 동적 점수
- `γ_i,j,t,r`: 번호쌍 상호작용
- `Z_r,t`: 모든 8,145,060개 조합의 확률 합을 1로 만드는 정규화항

번호쌍 항은 데이터 희소성이 매우 크므로 강한 수축 규제를 적용해야 한다. 검증되지 않은 번호쌍 효과를 확대하지 않는다.

---

## 7. 최종 동적 혼합분포

```text
P_t(S)
= w_0,t · P0(S)
+ w_1,t · P_1,t(S)
+ w_2,t · P_2,t(S)
+ w_3,t · P_3,t(S)
```

제약:

```text
w_r,t ≥ 0
Σ_r w_r,t = 1
```

각 모형의 의미:

### M0 — 무신호·균등

모든 조합이 동일한 확률이라는 기준모형.

### M1 — 지속

최근 또는 장기 신호가 다음 회차에도 유지된다는 가설.

### M2 — 반전

최근 과다출현·과소출현·집중 패턴이 평균회귀하거나 부호 전환된다는 가설.

### M3 — 구조변화

기존 생성구조가 무효화되고 새로운 상태가 시작됐다는 가설. 변화점, 계수 부호 전환, 불확실성 급증을 포함한다.

---

## 8. 모형 가중치 갱신

각 회차의 사전 예측 확률로만 가중치를 갱신한다.

```text
w_r,t ∝ (w_r,t-1 ^ δ) · P_r,t-1(Y_t-1 | D_t-2)
```

필수 제약:

- 한 번의 우연한 적중으로 가중치 급변 금지
- 학습률 제한
- 모델별 가중치 상한
- 최소 검증기간 적용
- 미래 데이터를 이용한 사후 조정 금지
- 비균등모형이 재현되지 않으면 M0 비중 확대

---

## 9. 무작위성 게이트

무작위성 게이트는 비균등모형을 활성화할 자격을 판단한다.

검토 대상:

- 번호별 균등성
- 회차 간 독립성
- 시차별 의존성
- 번호쌍 동시출현
- 국소 엔트로피 변화
- 변화점
- 미래 walk-forward 성능

원칙:

```text
신호 미확인 또는 재현 실패 → w_0,t 증가
```

무작위성 검정에서 작은 p값 하나가 나왔다고 예측모형을 활성화하지 않는다. 다중검정 보정과 미래 재현이 필요하다.

---

## 10. 불확실성

번호별 다음 회차 포함확률:

```text
p_i,t = P(i ∈ Y_t | D_t-1)
```

각 번호 및 조합은 점추정치뿐 아니라 사후분포를 가져야 한다.

사용자 표시:

- 번호별 포함확률과 95% 신용구간
- 조합별 균등 기준 대비 Lift와 95% 신용구간
- 예상 일치번호 수와 95% 예측구간
- 3개 이상 일치확률
- 모형별 현재 가중치

Lift:

```text
Lift_t(S) = P_t(S) / P0(S)
```

Lift의 95% 구간이 1을 포함하면 `통계적 우위 없음`으로 표시한다.

---

## 11. 모델 승격 기준

99.9%는 다음 번호가 맞을 확률이 아니다. 비균등모형이 균등모형과 다른 예측력을 가졌다고 인정하기 위한 연구 기준이다.

```text
P(비균등모형이 균등모형보다 우수 | 데이터) ≥ 99.9%
```

추가 필수조건:

- 미래 walk-forward 구간에서 Brier Score 개선
- Log Loss 개선
- Calibration 개선
- 다중검정 보정 통과
- 둘 이상의 독립 검증구간에서 재현
- 특정 구간이나 일부 성공 사례만 선택하지 않음

---

## 12. 최종 후보 5세트 최적화

후보 조합 `S_1 ... S_5`를 다음 목적함수로 선택한다.

```text
maximize
  Σ_(k=1..5) log P_t(S_k)
  - ρ · Σ_(a<b) |S_a ∩ S_b|
  - τ · Σ_(k=1..5) U(S_k)
  - κ · Σ_(k=1..5) C(S_k)
```

- `P_t(S_k)`: 최종 혼합확률
- `|S_a ∩ S_b|`: 후보 간 번호 중복
- `U(S_k)`: 추정 불확실성
- `C(S_k)`: 군중 선택 가능성

군중 회피항의 전체 영향은 최대 5%로 제한한다.

성공목표 우선순위:

1. 정확히 6개 적중 가능성
2. 5세트 중 최고 일치 개수
3. 후보 다양성
4. 군중 회피

정확히 6개 적중 가능성만 최우선으로 평가할 때는 확률 상위 5개의 서로 다른 조합을 사용한다. 중복 페널티는 2차 목표에만 제한적으로 적용한다.

---

## 13. 출력 계약

매 회차 출력:

- 6개 번호 조합 5세트
- 후보 순위
- 예측 대상 회차
- 당첨자 발표일
- 모델 버전
- 입력 데이터의 마지막 회차
- 생성시각
- 잠금상태
- 예측 해시
- `통계적 우위 없음` 또는 검증상태
- 상세정보는 기본 숨김

`예측하기` 버튼은 브라우저에서 새 결과를 생성하지 않는다. Python 엔진이 사전 생성하고 잠근 JSON을 공개한다.

## 14. 구현 불변조건

다음 테스트를 반드시 통과해야 한다.

1. 모든 조합확률은 음수가 아니다.
2. 각 모형 및 혼합분포의 전체 확률합은 허용 오차 내에서 1이다.
3. 동일 데이터·동일 버전·동일 시드에서 결과가 재현된다.
4. 예측 대상 회차 이후 데이터는 입력에 포함되지 않는다.
5. 신호가 없는 합성데이터에서 M0로 복귀한다.
6. 알려진 편향을 심은 합성데이터에서 해당 편향을 탐지할 수 있다.
7. 한 회차의 우연한 적중만으로 모형 가중치가 폭증하지 않는다.
8. 군중 회피항이 전체 목적함수에 5%를 초과해 영향을 주지 않는다.
