# Gate 1 검토 안내서

대상: 비개발자 검토자  
현재 범위: 데이터 무결성 및 과거번호 아카이브  
예측 엔진: Gate 2에서 개발

## 1. 이번 단계에서 확인할 내용

Gate 1은 다음 항목만 검토합니다.

1. 1회부터 1230회까지 회차가 빠짐없이 표시되는지
2. 각 회차의 추첨일·당첨번호 6개·보너스번호가 올바르게 보이는지
3. 번호가 구간별 원형 색상으로 표시되는지
4. 회차 검색이 작동하는지
5. 연도 필터가 작동하는지
6. 최신순·오래된순 정렬이 작동하는지
7. 데이터 검증상태와 체크섬을 상세화면에서 확인할 수 있는지
8. 번호별 통계가 예측값으로 오해되지 않도록 안내되는지

이번 단계에서 `예측하기` 버튼은 비활성 상태가 정상입니다.

## 2. 미리보기 실행 방법

GitHub Actions의 `gate1-archive-preview` ZIP을 내려받아 압축을 풉니다.

압축을 푼 폴더에서 명령 프롬프트 또는 터미널을 열고 다음 명령을 입력합니다.

```bash
python -m http.server 8000
```

웹브라우저 주소창에 다음 주소를 입력합니다.

```text
http://localhost:8000/app/archive.html
```

파일을 직접 더블클릭하면 브라우저 보안정책 때문에 JSON을 읽지 못할 수 있으므로 위 방식으로 실행해야 합니다.

## 3. 화면에서 확인할 기능

### 회차 검색

검색창에 `1230`을 입력하면 1230회 한 건이 표시되어야 합니다.

### 연도 필터

연도를 선택하면 해당 연도 회차만 표시되어야 합니다.

### 정렬

- `최신 회차부터`: 1230회부터 표시
- `1회부터`: 1회부터 표시

### 원형 번호 색상

- 1~10: 노랑
- 11~20: 파랑
- 21~30: 빨강
- 31~40: 회색
- 41~45: 초록

### 데이터 상세

각 회차의 `데이터 상세 보기`를 열면 다음을 확인할 수 있습니다.

- 데이터 출처
- 검증상태
- 잠금 여부
- 레코드 SHA-256 체크섬

## 4. 검증상태의 의미

### `공식 대조 완료`

동행복권 공식 응답과 추첨일·당첨번호 6개·보너스번호가 정확히 일치하고 잠금된 상태입니다.

### `자동 형식 검증`

다음 검사를 모두 통과했지만 공식 자동 대조가 완료되지 않은 상태입니다.

- 회차 연속성
- 추첨일 형식 및 주기
- 당첨번호 6개 여부
- 번호 중복 여부
- 1~45 범위
- 보너스번호 중복 여부
- 레코드 체크섬

이 상태를 공식 검증 완료로 표현하지 않습니다.

## 5. 현재 데이터 상태

- 범위: 1~1230회
- 전체 레코드: 1,230개
- 구조 및 체크섬 검사: 통과
- 동일 입력 재생성 SHA 검사: 통과
- 공식 자동 대조: 기존 동행복권 JSON 엔드포인트가 응답하지 않아 미완료
- 현재 상태: 1,230개 모두 `auto_checked`, 잠금 해제

따라서 현재 아카이브는 검토용입니다. 공식 검증 전략이 확정되기 전에는 `verified` 또는 `locked`로 표시하지 않습니다.

## 6. 1230회 표시값

```text
추첨일: 2026-06-27
당첨번호: 3, 8, 9, 22, 28, 42
보너스번호: 45
```

## 7. Gate 1 승인 시 함께 결정할 내용

다음 중 하나를 지정해야 Gate 2의 데이터 사용범위가 명확해집니다.

1. 공식 페이지 기반 전수 대조 후에만 Gate 2 시작
2. 복수 독립 출처 일치를 조건으로 연구용 백테스트를 허용하되 실제 미래예측 공개는 공식 잠금 이후 진행
3. 현재 `auto_checked` 데이터를 연구용 백테스트에만 사용하고 공식 검증은 별도 트랙으로 병행

현재 권장안은 **3번**입니다. 알고리즘 구현과 검증 연구는 진행하되, 실제 미래예측 공개에 사용되는 데이터는 공식 검증 후 잠그는 방식입니다.

## 8. 검토 결과 전달 방법

다음 항목을 중심으로 의견을 전달하면 됩니다.

- 번호 또는 날짜 오류
- 검색·필터·정렬 문제
- 번호 원형 디자인 문제
- 모바일 화면 문제
- 표시 문구가 어렵거나 오해될 수 있는 부분
- 위 데이터 사용정책 1·2·3 중 선택

Gate 1 승인 전에는 Draft PR을 `main`에 병합하지 않습니다.
