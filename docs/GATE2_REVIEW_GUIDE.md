# Gate 2-1 검토 안내서

대상: 비개발자 검토자  
현재 단계: 예측 엔진 명세 고정  
아직 하지 않은 작업: Python 구현, 전체 백테스트, 1231회 후보 생성

## 1. 이번 단계의 결론

이번 문서는 “어떤 번호를 추천할지”가 아니라 “번호를 추천하는 시스템이 어떤 규칙을 절대로 어기지 않아야 하는지”를 확정합니다.

확인할 핵심은 여섯 가지입니다.

1. 균등 무작위모형 M0가 항상 살아 있는가
2. 최근 신호의 지속 M1과 반전 M2를 동시에 경쟁시키는가
3. 구조변화 M3를 별도로 감지하는가
4. 신호가 없을 때 M0로 돌아가는가
5. 과거 데이터에서 잘 보였다는 이유만으로 검증 완료라고 하지 않는가
6. 공식 검증 전에는 실제 미래예측을 공개·잠그지 않는가

## 2. 엔진이 번호를 계산하는 방식

각 번호에 단순 출현횟수 하나를 붙이는 방식이 아닙니다.

- 장기 출현 편차
- 최근 10·30·52·104회 편차
- 단기와 중기의 추세 차이
- 미출현 간격
- 최근 구간과 이전 구간의 변화
- EWMA
- CUSUM

을 별도 신호로 계산합니다.

각 신호는 지속·반전·구조변화 모형에서 서로 다르게 해석됩니다. 실제 사전예측 성능이 없으면 해당 모형의 영향은 줄어듭니다.

## 3. 무작위성을 반영하는 방식

무작위성은 엔트로피 점수 하나가 아닙니다.

- 균등모형 M0를 기준으로 유지
- 비균등모형은 shadow 상태에서 먼저 평가
- 역사적 검증을 통과해도 최대 30%만 반영하는 CANDIDATE 상태
- 실제 미래 52회 이상 검증 전에는 PROMOTED 금지

즉, 복잡한 모형이 근거 없이 균등모형을 밀어내지 못합니다.

## 4. 오차와 성능을 확인하는 방식

가장 중요한 지표:

```text
Joint Log Loss
```

실제 당첨조합에 모델이 사전에 얼마나 적절한 확률을 부여했는지 평가합니다.

함께 확인:

- Brier Score
- Calibration
- 5세트 중 최고 일치번호
- 평균 일치번호
- 3개·4개·5개·6개 일치 빈도
- 후보 간 중복과 전체 커버리지

한 번 4개 또는 5개를 맞힌 사례만으로 검증을 주장하지 않습니다.

## 5. 백테스트 구간

```text
1~299회 데이터 → 300회 예측
...
1~1229회 데이터 → 1230회 예측
```

총 931개 회차를 Walk-forward 방식으로 평가합니다.

안정성 확인 구간:

- A: 300~609회
- B: 610~919회
- C: 920~1230회

결과가 좋은 구간만 다시 선택하지 않습니다.

## 6. 검증 기준

역사적 데이터에서 CANDIDATE가 되려면 다음을 모두 충족해야 합니다.

- 균등모형보다 Joint Log Loss 개선
- 개선확률 99.9% 이상
- 다중검정 보정 p값 0.001 이하
- Brier Score 악화 없음
- 세 구간 중 최소 두 구간에서 같은 방향
- 균등 합성데이터 오탐률 0.1% 이하

PROMOTED가 되려면 공식 검증 데이터와 최소 52회 실제 미래예측이 추가로 필요합니다.

## 7. 후보 5세트 선택 원칙

1순위는 정확히 6개가 일치할 확률입니다.

확률이 거의 같은 후보 안에서만 다음을 고려합니다.

- 최고 부분일치 가능성
- 후보 간 중복 감소
- 번호 커버리지
- 군중 회피

군중 회피 영향은 보조목표의 5% 이하입니다.

## 8. 승인 후 진행되는 작업

Gate 2-1 승인 후:

1. Python 엔진 골격
2. 피처 계산
3. 조합확률 정확 정규화
4. M0~M3 구현
5. 무작위성 gate 상태기계
6. 후보 5세트 최적화
7. 단위테스트와 합성 smoke test

전체 931회 백테스트는 그 다음 단계에서 실행합니다.
