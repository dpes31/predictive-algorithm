# Validation Protocol

## 1. 목적

모델이 과거 데이터의 우연한 패턴을 암기하지 않고, 균등 무작위 기준보다 재현 가능한 예측력을 갖는지 검증합니다.

## 2. Walk-forward 원칙

각 회차 t를 예측할 때 `t-1`회까지의 데이터만 사용합니다.

```text
1~299회 학습 → 300회 예측
1~300회 학습 → 301회 예측
...
1~1229회 학습 → 1230회 예측
```

미래 회차 데이터, 미래 결과에서 계산된 특징, 전체 기간을 본 뒤 선택한 하이퍼파라미터를 과거 예측에 소급 적용하지 않습니다.

## 3. 비교 기준모형

- M0 균등 무작위 5세트
- 누적 고빈도
- 최근 52회 고빈도
- 최근 52회 저빈도
- 장기 미출현
- 직전 회차 번호 재사용
- 단순 홀짝·구간 균형 조합

복잡한 모델은 최소한 이 기준들과 비교되어야 합니다.

## 4. 핵심 평가지표

### 확률예측

- Brier Score
- Log Loss
- Calibration
- 균등모형 대비 평균 점수 차이

### 조합 결과

- 세트당 평균 일치번호
- 5세트 중 최고 일치번호
- 3개 이상 일치 빈도
- 4개 이상 일치 빈도
- 5개 이상 일치 빈도
- 6개 적중 빈도

정확한 6개 적중은 희귀하여 단독 검증지표로 사용하지 않습니다.

## 5. 불확실성 및 승격

사용자 표시는 95% 신용구간 또는 예측구간을 사용합니다.

비균등모형 정식 승격은 다음 조건을 모두 요구합니다.

```text
P(비균등모형이 균등모형보다 우수 | 데이터) ≥ 99.9%
```

추가조건:

- 미래 walk-forward 성능 개선
- Brier Score, Log Loss, Calibration 중 사전 지정 지표 개선
- 다중검정 보정 통과
- 둘 이상의 독립 검증구간에서 방향 재현
- 모델 선택 횟수와 하이퍼파라미터 탐색을 기록

## 6. 합성데이터 검증

### Negative control

완전 균등 6/45 데이터를 반복 생성합니다.

기대결과:

- 비균등모형이 지속적으로 활성화되지 않음
- M0 가중치가 우세
- 정한 오탐률 이하

### Positive control

다음 편향을 의도적으로 심습니다.

- 특정 번호의 약한 출현확률 증가
- 특정 번호쌍 동시출현 증가
- 일정 회차 이후 구조변화
- 점진적 드리프트
- 일시적 이상치

기대결과:

- 편향의 존재와 방향을 탐지
- 변화가 사라지면 비균등 가중치 감소
- 탐지 지연시간과 미탐률 기록

## 7. 재현성

모든 실행은 다음을 기록합니다.

- 데이터 버전
- 모델 버전
- 코드 커밋 SHA
- 랜덤 시드
- 하이퍼파라미터
- 생성시각
- 입력 마지막 회차
- 결과 해시

동일 입력과 버전에서는 동일 결과가 재현되어야 합니다.

## 8. 금지되는 검증 방식

- 전체 데이터를 학습하고 같은 데이터에서 평가
- 가장 잘 나온 기간만 보고
- 실패한 모델·기간 삭제
- 실제 결과를 본 뒤 예측규칙 수정
- 여러 지표 중 유리한 지표만 선택
- 95% 구간이 1을 포함하는 Lift를 우위로 표현
- 한 번의 4개·5개 적중을 알고리즘 검증으로 주장

## 9. 예측 잠금

추첨 전에 다음을 고정합니다.

- 후보 5세트
- 모델 버전
- 데이터 버전
- 모형 가중치
- 생성시각
- 해시

추첨 후에는 자동 평가만 수행하며 잠금 결과를 수정하지 않습니다.
