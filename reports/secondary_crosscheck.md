# Gate 1 Secondary Cross-check Record

작성일: 2026-06-30  
목적: 공식 자동 대조가 불가능한 상태에서 수행한 보조 교차검증을 기록합니다.

## 1. 1~1229회 본번호 및 날짜

이전 대화에서 별도로 구축한 1~1229회 데이터셋과 현재 `data/draws.json`의 다음 필드를 비교했습니다.

- 회차
- 추첨일
- 당첨번호 6개

비교 결과:

```text
비교 회차: 1~1229
불일치: 0건
```

이전 데이터셋 파일의 SHA-256:

```text
b82cc4a42fb1488bdc5901ec3ab1eef2382fa921855c1c9cda36b875946b5a3a
```

주의: 이전 데이터셋은 보너스번호를 포함하지 않았으므로, 이 비교는 1~1229회의 보너스번호를 독립적으로 검증한 것이 아닙니다.

## 2. 1230회

현재 표시값:

```text
추첨일: 2026-06-27
당첨번호: 3, 8, 9, 22, 28, 42
보너스번호: 45
```

보조 확인 출처:

- 연합뉴스: https://www.yna.co.kr/amp/view/AKR20260627046200002
- 뉴시스: https://www.newsis.com/view/NISX20260627_0003686246

두 보도 모두 동행복권 발표를 인용해 동일한 본번호와 보너스번호를 제시합니다.

## 3. 구조 검증

1~1230회 전체에 대해 다음 검사를 통과했습니다.

- 결측 회차 없음
- 중복 회차 없음
- 본번호 정확히 6개
- 본번호 중복 없음
- 번호 범위 1~45
- 보너스번호가 본번호와 중복되지 않음
- 추첨일 ISO 형식
- 추첨일 토요일 여부
- 인접 회차 7일 간격
- 레코드별 SHA-256 일치
- 아카이브 파생 데이터가 원본과 일치

## 4. 현재 판정

보조 교차검증은 데이터 오류 가능성을 낮추지만 공식 검증을 대체하지 않습니다.

동행복권의 기존 JSON 엔드포인트가 자동 요청에 응답하지 않았기 때문에 현재 1~1230회는 모두 다음 상태로 유지합니다.

```text
verification_status: auto_checked
locked: false
```

공식 자동 대조 또는 승인된 수동 공식 대조가 완료되기 전에는 `verified` 또는 `locked`로 표시하지 않습니다.
