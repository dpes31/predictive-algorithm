# Gate 2-3 합성 검증 결과

## 1. 결론

**Gate 2-3은 통과하지 못했습니다.**

균등 무작위 데이터에서의 관측 오탐률은 낮았지만, 사전지정한 positive-control 대부분에서 기대한 모형이 신호를 탐지하지 못했습니다. 따라서 실제 1~1230회 데이터의 Walk-forward 백테스트인 Gate 2-4로 이동하지 않습니다.

이 결과는 삭제하거나 유리하게 재해석하지 않고 실패 결과로 보존합니다.

## 5. 발견된 구조적 문제

### A. M1·M2 점수 크기 과대

현재 명세는 표준화 피처 `z`를 그대로 조합 logits에 사용합니다. 약한 실제 신호에도 `η=±z`가 지나치게 확신적인 확률분포를 만들어, 방향이 맞더라도 proper score가 악화될 수 있습니다.

필요 조치:

- 피처 방향과 효과크기를 분리
- 사전고정된 shrinkage 또는 temperature 계수 도입
- 계수별 sub-expert를 합성 null에서 다시 캘리브레이션

이는 하이퍼파라미터·sub-expert 변경이므로 사용자 승인과 모델 MINOR 버전 증가가 필요합니다.

### B. M3 진단의 상한 포화

`z_shift_52`, `z_shift_104`, CUSUM을 `[-3,+3]`으로 제한한 뒤 최대값으로 null calibration을 수행해, 99.9% null 분위수가 모두 3.0에 포화됐습니다. 관측값도 3.0을 넘을 수 없어 p-value가 충분히 작아질 수 없습니다.

필요 조치:

- 예측모형에 들어가는 피처는 기존처럼 winsorize 유지
- **변화 탐지용 global diagnostic만 winsorize 이전 raw statistic으로 별도 계산**
- raw diagnostic의 familywise null 분포 재산출

이는 M3 활성화 규칙의 명세 보정이므로 사용자 승인이 필요합니다.

### C. 번호쌍 진단의 탐지력 부족

104회 window에서 990개 번호쌍과 여러 시점을 동시에 보정한 최대통계량 기준은 매우 보수적입니다. null 99.9% 최대 z값은 약 8.37이었고, factor 3.0 신호도 이 기준을 넘지 못했습니다.

필요 조치 후보:

- pair interaction은 계속 비활성 유지
- 장기 window를 별도 진단으로 추가하거나
- 사전지정 pair와 탐색적 전체 pair의 검정체계를 분리

새 window 또는 진단 추가는 feature contract 변경과 사용자 승인이 필요합니다.

### D. M3 승리율 표시 오류

구조변화 실험에서 M3 승리율 100%가 기록됐지만, M3의 평균 개선점수는 정확히 0이고 탐지율도 0%였습니다. M1·M2 점수가 음수여서 비활성 M3가 상대적으로 1위가 된 것입니다.

따라서 향후 `승리`는 다음을 모두 충족할 때만 인정해야 합니다.

- 기대모형의 Δ Log Loss > 0
- 비교모형보다 엄격히 우수
- 해당 진단 gate 활성화

이는 보고 로직 오류 수정이며 기존 원결과는 그대로 보존합니다.
